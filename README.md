Общие системные улучшения

Разделение ответственности (SRP): Вся логика вынесена из функции main в специализированные модули.

Безопасный файловый ввод: Добавлены строгие проверки на существование файлов и корректность структуры данных внутри них.

Валидация данных: Внедрены функции IsNumber и SafeReadInt для предотвращения ошибок при наличии посторонних символов в исходных файлах.

Соответствие стандартам: Удален using namespace std, переменные переименованы в соответствии с их смысловым значением (CamelCase/PascalCase).

Подробный разбор по задачам

Задание 1 — Оптимальный выбор артефактов (Рюкзак)

Задача: Выбрать предметы с суммарным весом в диапазоне $[A, B]$ при минимальном объеме.

Было: Неинформативные имена переменных (c, d, dp), отсутствие проверки открытия файла, использование and вместо &&.

Стало:Реализован алгоритм динамического программирования с использованием вектора MinWeight.

Исправлена ошибка, при которой программа выводила : вместо сообщения «Решение отсутствует».

Добавлена возможность восстановления состава предметов через обратный ход по таблице весов.

Задание 2 — Черепашка (Максимальный путь)

Задача: Найти путь с максимальной суммой от правого верхнего до левого нижнего угла доски.

Было: Вся логика находилась в main, отсутствовали проверки границ массива $N < 80$.

Стало:

Оптимизация памяти: Для расчета используется вспомогательная матрица ResultBoard, где ResultBoard[i][j] хранит лучшую сумму до данной клетки.

Дружественный интерфейс: Добавлен вывод итоговой суммы в файл 22.txt и информационные сообщения в консоль. 

Исправлена логика движения — теперь алгоритм строго следует правилу «только влево или вниз».

Задание 3 — К-ичные числа (Комбинаторика)Задача: Посчитать количество $N$-разрядных чисел в системе $K$, содержащих более трех нулей подряд.Было: Нечитаемые переменные (nz, oz, tz, thz), отсутствие форматирования кода, не вынесен общий множитель в формулах.Стало:Логика состояний: Введены переменные NoZeroCount, OneZeroCount и т.д., отражающие текущее состояние цепочки нулей.Математическая оптимизация: Общий множитель $(K-1)$ вынесен за скобки, что упростило вычисления и улучшило читаемость формул.Точность: Все расчеты переведены на тип float/double для предотвращения переполнения при больших $N$ (до 20).

Задание 4 — Разбор логического выраженияЗадача: Вычислить значение рекурсивного выражения (And, Or, T, F).Критическое исправление: Согласно условию, в коде запрещено использование циклов.Было: Наличие циклов while для пропуска пробелов и обработки строк.Стало:Рекурсивный парсер: Создана функция ParseExpression, которая вызывает саму себя для вычисления вложенных операторов And(...) и Or(...).Рекурсивная проверка: Функция CheckSpaces теперь рекурсивно проверяет строку на наличие запрещенных пробелов без использования циклов.Обработка ошибок: Если структура выражения нарушена, программа корректно сообщает об ошибке позиции.
